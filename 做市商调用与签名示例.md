# Mock Trade 做市商调用与 HMAC 签名示例

> 适用：做市商调用 Mock Trade（做市对敲成交）接口。
> 签名/校验逻辑来自 `unimargin-http-gateway` 的 `ApiAuthInterceptor.apikeyAuth`，与服务端一致。

---

## 1. 接入入口（关键）

| 项 | 值 |
| --- | --- |
| **Host（必须）** | `api.bifu.internal`（system 网关，**内部域名**，MM 鉴权仅此生效） |
| 现货路由 | `POST /api/v1/private/spot/mockTrade` |
| 合约路由 | `POST /api/v1/private/contract/mockTrade` |
| Content-Type | `application/json` |

> ⚠️ 必须用 `api.bifu.internal` 这个 Host（做市机内网 DNS/hosts 解析到 envoy-gw）。用公网域名（`api.bifu.dev`/`api.bifu.co` 等）会落到普通网关，恒返回 `PERMISSION_DENIED`。

---

## 2. 鉴权 Header

| Header | 含义 |
| --- | --- |
| `Decode-MM-Auth-Access-Key` | 做市账户 accessKey（运营在 Nacos `market_maker_config.json` 下发） |
| `Decode-MM-Auth-Timestamp` | 当前时间戳，**毫秒**（13 位）。与服务器时间偏差需 **≤ ±30 秒** |
| `Decode-MM-Auth-Signature` | 见下方签名算法（HMAC-SHA256，小写 hex） |

---

## 3. 签名算法

```
signMessage = <requestURI> + "|" + <timestamp>
signature   = HMAC_SHA256( key = secretKey, message = signMessage )   // 输出小写 hex
```

- `requestURI`：**只含路径，不含 Host、不含 query**。例如 `/api/v1/private/spot/mockTrade`。
- `timestamp`：与 `Decode-MM-Auth-Timestamp` header **完全相同**的毫秒时间戳。
- `secretKey`：与 accessKey 配对的密钥。
- **签名只覆盖 `路径|时间戳`，不包含 body**（与服务端实现一致）。
- 分隔符是竖线 `|`。

示例 signMessage：
```
/api/v1/private/spot/mockTrade|1717113600000
```

## 4. 请求体（`MockTradeParam`）

```json
{
  "symbolId": "10000001",
  "price": "65000.5",
  "quantity": "0.01",
  "side": "BUY",
  "clientId": "mm-20260603-0001"
}
```

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `symbolId` | 是 | **数字字符串**交易对 ID（非 "BTC_USDT"） |
| `price` | 是 | 期望价（字符串），服务端按 `[bid1, ask1]` 钳制 |
| `quantity` | 是 | 数量（base，字符串） |
| `side` | 是 | `BUY` / `SELL`（taker 视角） |
| `clientId` | 否 | 幂等键（v1 暂不去重） |

### 响应

```json
{ "code": 0, "data": {

  "executedPrice": "65000.5",

```

## 5. 示例代码

### 5.1 Bash（curl + openssl）

```bash
#!/usr/bin/env bash
set -euo pipefail

HOST="https://api.bifu.internal"
ACCESS_KEY="YOUR_ACCESS_KEY"
SECRET_KEY="YOUR_SECRET_KEY"
PATH_URI="/api/v1/private/spot/mockTrade"     # 合约用 /api/v1/private/contract/mockTrade
TS="$(date +%s%3N)"                            # 13 位毫秒时间戳

SIGN_MSG="${PATH_URI}|${TS}"
SIG="$(printf '%s' "${SIGN_MSG}" | openssl dgst -sha256 -hmac "${SECRET_KEY}" | sed 's/^.*= //')"

curl -sS -X POST "${HOST}${PATH_URI}" \
  -H "Content-Type: application/json" \
  -H "Decode-MM-Auth-Access-Key: ${ACCESS_KEY}" \
  -H "Decode-MM-Auth-Timestamp: ${TS}" \
  -H "Decode-MM-Auth-Signature: ${SIG}" \
  -d '{"symbolId":"10000001","price":"65000.5","quantity":"0.01","side":"BUY","clientId":"mm-001"}'
```

> 注意 `printf '%s'`（不加换行），否则 message 末尾多个 `\n` 会签名失败。
> `date +%s%3N` 在部分系统（如 macOS 自带 date）不支持毫秒，可改 `python3 -c 'import time;print(int(time.time()*1000))'`。

### 5.2 Python

```python
import hashlib, hmac, time, requests

HOST = "https://api.bifu.internal"
ACCESS_KEY = "YOUR_ACCESS_KEY"
SECRET_KEY = "YOUR_SECRET_KEY"
PATH = "/api/v1/private/spot/mockTrade"     # 合约: /api/v1/private/contract/mockTrade

ts = str(int(time.time() * 1000))           # 毫秒
sign_msg = f"{PATH}|{ts}"
sig = hmac.new(SECRET_KEY.encode("utf-8"), sign_msg.encode("utf-8"), hashlib.sha256).hexdigest()

headers = {
    "Content-Type": "application/json",
    "Decode-MM-Auth-Access-Key": ACCESS_KEY,
    "Decode-MM-Auth-Timestamp": ts,
    "Decode-MM-Auth-Signature": sig,
}
body = {"symbolId": "10000001", "price": "65000.5", "quantity": "0.01",
        "side": "BUY", "clientId": "mm-001"}

resp = requests.post(HOST + PATH, json=body, headers=headers, timeout=5)
print(resp.status_code, resp.text)
```

### 5.3 Go

```go
package main

import (
	"bytes"
	"crypto/hmac"
	"crypto/sha256"
	"encoding/hex"
	"fmt"
	"io"
	"net/http"
	"strconv"
	"time"
)

func main() {
	host := "https://api.bifu.internal"
	accessKey := "YOUR_ACCESS_KEY"
	secretKey := "YOUR_SECRET_KEY"
	path := "/api/v1/private/spot/mockTrade" // 合约: /api/v1/private/contract/mockTrade

	ts := strconv.FormatInt(time.Now().UnixMilli(), 10) // 毫秒
	signMsg := path + "|" + ts
	mac := hmac.New(sha256.New, []byte(secretKey))
	mac.Write([]byte(signMsg))
	sig := hex.EncodeToString(mac.Sum(nil))

	body := []byte(`{"symbolId":"10000001","price":"65000.5","quantity":"0.01","side":"BUY","clientId":"mm-001"}`)
	req, _ := http.NewRequest(http.MethodPost, host+path, bytes.NewReader(body))
	req.Header.Set("Content-Type", "application/json")
	req.Header.Set("Decode-MM-Auth-Access-Key", accessKey)
	req.Header.Set("Decode-MM-Auth-Timestamp", ts)
	req.Header.Set("Decode-MM-Auth-Signature", sig)

	resp, err := http.DefaultClient.Do(req)
	if err != nil {
		panic(err)
	}
	defer resp.Body.Close()
	b, _ := io.ReadAll(resp.Body)
	fmt.Println(resp.StatusCode, string(b))
}
```

### 5.4 Java（与服务端 Guava 实现一致，便于自测对账）

```java
import com.google.common.hash.Hashing;
import java.nio.charset.StandardCharsets;

String path = "/api/v1/private/spot/mockTrade";
long ts = System.currentTimeMillis();
String signMsg = path + "|" + ts;
String sig = Hashing.hmacSha256(secretKey.getBytes(StandardCharsets.UTF_8))
        .hashString(signMsg, StandardCharsets.UTF_8)
        .toString(); // 小写 hex
// 然后用任意 HTTP 客户端带上 3 个 Decode-MM-Auth-* header 发 POST
```

## 6. 失败排查

| 现象 | 原因 |
| --- | --- |
| 401 / `PERMISSION_DENIED`（无任何业务码） | 没走 system 网关（Host 不是 `api.bifu.internal`）；或 accessKey 不存在；或签名/时间戳错 |
| `GATEWAY_MM_PERMISSION_DENIED` | accessKey 解析不到 / 身份不匹配 / IP 不在白名单 |
| `GATEWAY_MM_ACCOUNT_TYPE_MISMATCH` | 用现货 accessKey（accountType=1）打了合约接口（要 0），或反之 |
| 429 | 超过 10 次/秒/账户 限频 |
| 时间戳类报错（鉴权失败） | 本地时钟与服务器偏差 > ±30s，请校时（NTP） |
| `INVALID_CONTRACT` | `symbolId` 不存在/已下架 |
| `MOCK_TRADE_INVALID_PARAM` | price/quantity ≤ 0 或缺失 |
| `NO_REFERENCE_PRICE` / `MOCK_TRADE_PRICE_DEVIATION` | 盘口双边为空且无 lastPrice / 入参价偏离 lastPrice 超 ±5% |

## 7. 注意事项

1. **时钟同步**：时间戳毫秒，窗口 ±30s，务必 NTP 校时。
2. **IP 白名单**：accessKey 绑定 `allowIps`，做市机出口 IP（envoy 透传的 `X-Forwarded-For` 首个有效 IP）必须在内。
3. **签名只覆盖 `路径|时间戳`**：同一秒内不同 body 的签名相同——这是当前实现；幂等用 `clientId` 控制。
4. **路径区分现货/合约**：签名里的 path 要与实际请求 path 一致。
5. **限频 10/s/账户**，多数币对远低于此；主币按需控制节奏。
